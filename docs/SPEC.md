# 사진 정리 프로그램 (Photo Organizer) 스펙 문서

> 네트워크 드라이브의 대용량 사진 라이브러리를 스캔하여 중복·유사 사진을 검출하고, 종류별(스크린샷/문서/흔들린 사진 등)로 자동 분류하는 Windows GUI 애플리케이션

| 항목 | 내용 |
|------|------|
| 문서 버전 | 1.0 |
| 대상 플랫폼 | Windows 10/11 (64bit) + macOS (동등 지원) |
| 예상 규모 | 사진 10만 장 이상 |
| 분류 방식 | 하이브리드 (규칙 기반 + 딥러닝) |
| 개발 언어 | Python 3.11+ |

---

## 1. 개요 및 목표

### 1.1 배경
개인/조직이 네트워크 드라이브(NAS, 공유 폴더)에 축적한 수십만 장의 사진은 시간이 지나며 중복 파일, 거의 동일한 연사 사진, 잘못 찍힌 사진, 용도가 다른 이미지(스크린샷·문서 촬영)가 뒤섞인다. 이를 수동으로 정리하는 것은 사실상 불가능하다.

### 1.2 목표
- 대용량(10만 장 이상) 사진을 **안정적으로** 스캔한다.
- **완전 중복**(바이트 동일) 및 **유사 사진**(리사이즈·편집·연사)을 검출한다.
- 사진을 **종류별로 자동 분류**한다: 스크린샷, 문서/영수증, 흔들린(블러) 사진, 일반 사진 등.
- 사용자가 결과를 검토하고 **안전하게** 삭제/이동/정리하도록 GUI를 제공한다.

### 1.3 핵심 원칙
1. **비파괴성 우선** — 프로그램은 기본적으로 파일을 삭제하지 않는다. 사용자의 명시적 확인 후에만 실행하며, 삭제 대신 휴지통/격리 폴더 이동을 기본값으로 한다.
2. **재개 가능성** — 10만 장 스캔은 수 시간이 걸릴 수 있다. 중단 후 재시작 시 이미 처리한 부분은 건너뛴다.
3. **응답성** — 무거운 작업 중에도 UI가 멈추지 않는다.

## 3. 주의해야 할 점 (리스크 및 대응)

이 프로젝트의 성패는 대부분 아래 항목에서 갈린다.

### 3.1 네트워크 드라이브 관련
- **속도 병목**: 네트워크 I/O는 로컬 대비 10~100배 느리다. 파일 전체를 읽는 해시 계산이 가장 큰 비용. → 크기 기준 사전 필터링으로 해시 대상 최소화, 유사도용 썸네일/해시는 한 번만 계산 후 캐시.
- **연결 끊김**: NAS 절전, Wi-Fi 불안정으로 중간에 끊길 수 있다. → 파일 단위 재시도(backoff), 실패 항목은 "보류"로 표시하고 계속 진행.
- **경로 길이 제한**: Windows 기본 260자(MAX_PATH) 제한. → `\\?\` 접두어 사용 또는 긴 경로 지원 옵션.
- **자격증명 보안**: 비밀번호를 평문 저장 금지. → Windows Credential Manager 사용 권장, 저장 안 함을 기본값으로.

### 3.2 데이터 안전성 (가장 중요)
- **오삭제 방지**: 유사 검출은 확률적이며 오탐(false positive)이 반드시 발생한다. 절대 자동 삭제하지 않는다.
- **원본 보호**: 삭제는 즉시 물리 삭제가 아닌 휴지통 또는 격리 폴더 이동을 기본으로. 복구 경로 확보.
- **중복 그룹의 "원본" 판정**: 어느 것을 남길지 잘못 정하면 원본을 지우고 열화본을 남기는 사고 발생. → 해상도·파일크기·촬영일(EXIF)·경로 우선순위로 자동 추천하되 최종 결정은 사용자.

### 3.3 이미지 포맷/메타데이터
- **HEIC/RAW**: iPhone HEIC, 카메라 RAW는 기본 라이브러리로 안 열릴 수 있다. → `pillow-heif`, `rawpy` 등 별도 처리 및 미지원 포맷 graceful skip.
- **EXIF 회전**: 썸네일·해시 계산 시 EXIF Orientation 미반영하면 같은 사진을 다르게 인식. → 로드 시 정규화.
- **손상 파일**: 잘린 JPEG 등. → 예외 처리 후 "손상" 카테고리로 분류.

### 3.4 분류 정확도
- 규칙 기반은 빠르지만 애매한 경우(예: 어두운 실내 사진 vs 흔들린 사진)에서 오류. → 딥러닝 모델로 2차 확인하는 하이브리드. 신뢰도 낮은 항목은 "검토 필요"로 별도 표시.
- 블러 판정 임계값은 이미지마다 편차가 크다. → 사용자 조정 가능한 슬라이더 제공, 미리보기로 즉시 확인.

### 3.5 규모 관련
- 10만 장 = 파일 목록만으로도 상당한 메모리. UI에 전부 렌더링 불가. → DB 저장 + 가상 스크롤(보이는 만큼만 썸네일 로드).
- 썸네일 10만 개 = 디스크 캐시 관리 필요. → 캐시 크기 상한 및 LRU 정리.

## 4. 기획 / 아키텍처 설계

### 4.1 기술 스택

| 영역 | 선택 | 사유 |
|------|------|------|
| 언어 | Python 3.11+ | 이미지 처리 생태계 풍부 |
| GUI | **PySide6 (Qt)** | 대량 이미지 UI, 가상 스크롤, 스레드 통합 우수 |
| 이미지 처리 | Pillow + pillow-heif, OpenCV | 로드/변환/블러 검출 |
| 유사도 해시 | imagehash (pHash/dHash) | 검증된 perceptual hash |
| 딥러닝 | ONNX Runtime + MobileNetV3/경량 분류 모델 | CPU에서 빠른 추론, .exe 내장 용이 |
| 저장소 | SQLite (WAL 모드) | 파일 메타/해시/결과 영속화, 재개 지원 |
| 병렬 처리 | multiprocessing (해시/추론), QThread (UI 연동) | CPU 바운드 병렬화 |
| 패키징 | PyInstaller | 단일 exe 배포 |

### 4.2 시스템 구성

```
┌─────────────────────────────────────────────┐
│                  GUI (PySide6)               │
│  스캔 설정 · 진행률 · 썸네일 그리드 · 결과 검토 │
└───────────────┬─────────────────────────────┘
                │ 시그널/슬롯 (스레드 안전)
┌───────────────▼─────────────────────────────┐
│              Application Core                 │
│  ScanController · JobQueue · ResultManager    │
└───┬───────────┬───────────┬───────────┬──────┘
    │           │           │           │
┌───▼───┐  ┌────▼────┐  ┌───▼────┐  ┌───▼─────┐
│Scanner│  │ Hasher  │  │Classifier│ │ Storage │
│(walk) │  │(중복/유사)│ │(규칙+DL) │ │(SQLite) │
└───────┘  └─────────┘  └─────────┘  └─────────┘
                              │
                        ┌─────▼─────┐
                        │Thumbnail  │
                        │Cache      │
                        └───────────┘
```

### 4.3 처리 파이프라인 (단계별)

```
[1] 디스커버리    : 디렉토리 재귀 walk → 이미지 경로 + 크기/수정시각 DB 기록 (빠름, I/O 최소)
        ↓
[2] 완전중복 검출 : 동일 크기 그룹만 해시(SHA-256) → 중복 그룹 확정
        ↓
[3] 지각해시 계산 : 각 이미지 pHash/dHash + 썸네일 생성 → DB 저장 (무거움, 병렬)
        ↓
[4] 유사군 클러스터: 해밍 거리 임계값으로 유사 그룹핑 (BK-tree/근사 이웃)
        ↓
[5] 분류          : 규칙 기반 1차 → 애매한 것만 DL 모델 2차 (하이브리드)
        ↓
[6] 결과 집계     : 카테고리/그룹별 정리, 대표 원본 추천
        ↓
[7] 사용자 검토   : GUI 표시 → 삭제/이동/정리 실행
```

각 단계는 DB에 상태를 기록하여 **어느 단계에서 멈춰도 재개** 가능하다.

### 4.4 분류 로직 (하이브리드)

**규칙 기반 1차 (빠른 필터)**
- **스크린샷**: EXIF에 카메라 제조사/모델 없음 + 해상도가 알려진 화면 비율(16:9, 화면 해상도군) + 파일명 패턴(`Screenshot`, `화면 캡처`, `IMG` 아님)
- **문서/영수증**: 흰 배경 비율 높음, 강한 직선 엣지(문서 테두리), OCR로 텍스트 밀도 높음
- **흔들린 사진**: 라플라시안 분산(선명도) < 임계값. 어두운 사진과 구분 위해 밝기·엣지 함께 판단
- **일반 사진**: 위에 해당 안 됨

**딥러닝 2차 (신뢰도 낮은 케이스만)**
- 규칙 신뢰도가 임계 이하이거나 규칙 간 충돌 시에만 MobileNet 계열 분류 모델 호출 → 속도와 정확도 균형

### 4.4a 베스트샷 선정 (유사 그룹 내 "제일 잘 나온 사진")

유사 그룹이 확정되면, 그룹 내 각 사진에 **품질 점수**를 매겨 최고점을 베스트샷으로 추천한다. "잘 나온"은 주관적이므로 여러 지표를 가중 합산하고, 최종 결정권은 사용자에게 둔다.

**품질 지표 (규칙 기반 — 빠름)**

| 지표 | 측정 방법 | 설명 |
|------|-----------|------|
| 선명도 | 라플라시안 분산 | 초점·흔들림. 블러 검출 엔진 재활용 |
| 노출 | 히스토그램 분석 | 언더/오버 노출(하얗게 날아감·뭉개짐) 감점 |
| 대비 | RMS 대비 | 밋밋한 사진 감점 |
| 눈 감음 | 얼굴/눈 검출 | 인물 그룹에서 눈 뜬 컷 우선 (연사에 중요) |

**미적 점수 (딥러닝 — 보강)**
- NIMA(Neural Image Assessment) 계열 경량 모델(ONNX)로 "사람이 좋다고 느낄 확률"을 점수화
- 규칙 점수만으로 순위가 애매하게 근접한 경우 위주로 호출해 비용 절감

**합산 및 표시**
```
그룹 내 각 사진: 최종점수 = Σ(지표점수 × 가중치)
    ↓
최고점 사진 → ⭐ 베스트샷, GUI에서 크게/맨 앞 배치
나머지 → 점수순 정렬
    ↓
선정 근거 표시 (예: "선명도 1위 · 눈 뜸 · 노출 양호")
```

**가중치 프리셋**
- 인물 프리셋: 눈 감음·선명도 비중 ↑
- 풍경 프리셋: 대비·미적 점수 비중 ↑
- 사용자가 설정에서 직접 조정 가능

**주의**: 자동 선정은 추천일 뿐이며, 사용자가 그룹 내 다른 사진을 클릭 한 번으로 베스트샷으로 교체할 수 있어야 한다(FR-07a).


### 4.5 데이터 모델 (SQLite 핵심 테이블)

```
files
  id, path, size, mtime, format,
  content_hash,          -- 완전중복용 (nullable, 필요시 계산)
  phash, dhash,          -- 유사도용
  thumb_path,            -- 썸네일 캐시 경로
  category,              -- screenshot/document/blurry/normal/corrupt
  category_confidence,
  scan_status,           -- discovered/hashed/classified/error
  error_msg

duplicate_groups
  group_id, file_id, is_representative

similar_groups
  group_id, file_id, similarity_score,
  quality_score,        -- 베스트샷 선정용 최종 점수
  is_best_shot,         -- 그룹 내 베스트샷 여부 (사용자 변경 가능)
  quality_detail        -- 지표별 점수 근거 (JSON: 선명도/노출/대비/눈감음/미적)

scan_sessions
  id, root_path, started_at, finished_at, total, processed, status

action_log            -- 감사/복구용
  id, file_id, action, from_path, to_path, timestamp
```

## 5. 구현 계획 (단계별 로드맵)

### Phase 1 — 코어 엔진 (GUI 없이 CLI로 검증)
- 디렉토리 스캔 + SQLite 스키마
- 완전 중복 검출 (크기 → 해시)
- EXIF 정규화 포함 이미지 로더 (HEIC/RAW 대응)
- 산출물: CLI로 중복 리포트 생성

### Phase 2 — 유사도 & 분류
- perceptual hash + 유사 그룹 클러스터링
- 규칙 기반 분류기 (스크린샷/문서/블러)
- 임계값 설정 파일

### Phase 3 — 딥러닝 하이브리드
- ONNX 경량 분류 모델 통합
- 규칙↔DL 신뢰도 기반 라우팅
- 썸네일 캐시 시스템
- **베스트샷 선정**: 품질 지표(선명도·노출·대비·눈감음) 계산 + NIMA 미적 점수, 가중 합산 및 근거 생성

### Phase 4 — GUI
- PySide6 메인 윈도우, 스캔 설정 화면
- 백그라운드 워커(QThread) + 진행률/ETA
- 가상 스크롤 썸네일 그리드
- 그룹/카테고리 검토 및 선택 UI

### Phase 5 — 안전 작업 & 마감
- 휴지통/격리 이동, 대표 원본 추천, 실행 로그
- 재개(증분 스캔), 오류 복구
- CSV/JSON 내보내기
- PyInstaller 패키징 + 설치 안내

### 5.1 성능 최적화 포인트
- 해시/추론은 `multiprocessing`으로 CPU 코어 수만큼 병렬
- 네트워크 파일은 한 번 읽을 때 해시+썸네일+분류 특징을 동시에 추출(재접근 최소화)
- DB는 배치 커밋(수백 건 단위), WAL 모드
- 썸네일은 지연 로딩 + 메모리 LRU 캐시

## 6. 테스트 계획

### 6.1 단위 테스트
| 대상 | 검증 내용 |
|------|-----------|
| 해시 계산 | 동일 파일 → 동일 해시, 1바이트 변경 → 다른 해시 |
| perceptual hash | 리사이즈·재압축본이 임계 내 유사로 판정되는지 |
| EXIF 정규화 | 회전 메타데이터 있는 이미지의 해시 일관성 |
| 블러 판정 | 선명/블러 샘플셋에 대한 정오 판정 |
| 포맷 로더 | HEIC/RAW/손상 파일 처리 및 graceful skip |

### 6.2 통합 테스트
- 알려진 정답이 있는 **테스트 데이터셋** 구성: 중복 N쌍, 유사 M군, 카테고리별 K장, 손상 파일 포함
- 파이프라인 전체 실행 → 정밀도(precision)/재현율(recall) 측정
- 분류 목표치 예시: 스크린샷 정밀도 ≥ 0.95, 블러 재현율 ≥ 0.85 (튜닝 기준선)
- **베스트샷 검증**: 사람이 라벨링한 "베스트" 정답과 자동 선정 결과의 일치율(Top-1 정확도) 측정. 연사·인물 그룹에서 눈 감은 컷이 베스트로 뽑히지 않는지 확인

### 6.3 성능/부하 테스트
- 10만 장 규모 합성 데이터셋으로 스캔 시간, 메모리 피크, DB 크기 측정
- 네트워크 드라이브 환경에서 처리 속도 및 끊김 복구 검증

### 6.4 안정성/복구 테스트
- 스캔 중 강제 종료 → 재시작 시 정상 재개 확인
- 네트워크 끊김 시뮬레이션 → 실패 항목 보류 및 나머지 진행 확인
- 삭제/이동 후 로그 기반 복구 확인

### 6.5 사용성 테스트
- 대량 그룹 검토 시 UI 응답성(프리징 없음)
- 오삭제 방지 흐름(확인 다이얼로그, 휴지통 기본값) 검증

## 7. 향후 확장 (Out of Scope, 참고)
- 얼굴/인물 기반 그룹핑
- 지오태그(위치) 기반 앨범 자동 생성
- Linux 지원 (Windows/macOS는 1급 지원으로 승격)
- 클라우드 스토리지(구글포토, OneDrive) 연동

> **크로스플랫폼 참고**: 플랫폼 의존 처리(긴 경로 `\\?\` 접두어, UNC/`/Volumes`
> 마운트, 시스템 폴더 스킵)는 `core/platform_utils.py`에 격리되어 있다. GUI(Phase 4)와
> 패키징(Phase 5)도 두 OS를 동등 타깃으로 한다: PyInstaller는 Windows `.exe`,
> macOS `.app`을 각각 빌드한다.

## 부록 B. 용어
- **완전 중복**: 바이트 단위로 동일한 파일
- **유사 사진**: 리사이즈·재압축·경미한 편집·연사 등으로 시각적으로 거의 같은 사진
- **perceptual hash**: 이미지 내용을 요약한 지문. 유사도를 해밍 거리로 측정
- **라플라시안 분산**: 이미지 선명도 지표. 낮을수록 흐림/흔들림
